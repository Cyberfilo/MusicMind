**`async def fetch_spotify_top_tracks_for_period(access_token: str, *, period: str = "month", limit: int = 50) -> list[dict[str, Any]]:`**
- Map period to Spotify time_range using SPOTIFY_TIME_RANGE_MAP. Raise ValueError if period not in map.
- Paginate with offset up to `limit` total tracks (single page is fine for stats; max 50 per page, usually want top 20-50).
- GET `{SPOTIFY_API_BASE}/me/top/tracks` with `time_range`, `limit`, `offset` params.
- Handle 403 (dev mode restriction) by logging warning and returning what we have.
- Return list of dicts with keys: `rank` (1-based position), `name`, `artist_name`, `album_name`, `duration_ms`, `catalog_id`, `preview_url`.
- The rank is derived from position in the response (Spotify returns in ranked order).
- Use same httpx.AsyncClient pattern and error handling as taste/fetch.py.

**`async def fetch_spotify_top_artists_for_period(access_token: str, *, period: str = "month", limit: int = 50) -> list[dict[str, Any]]:`**
- Same time_range mapping as above.
- GET `{SPOTIFY_API_BASE}/me/top/artists` with `time_range`, `limit` params.
- Handle 403 gracefully.
- Return list of dicts with keys: `rank` (1-based position), `name`, `id`, `genres`.

**`def compute_apple_music_top_tracks(songs: list[dict[str, Any]], *, period: str = "month") -> list[dict[str, Any]]:`**
- Per D-03, D-05: Apple Music has no native time-range API. Compute from cached library data.
- Filter songs by `date_added_to_library` based on period:
  - "month": added within last 30 days
  - "6months": added within last 180 days
  - "alltime": no filter
- Sort by `date_added_to_library` descending (most recently added first) as a proxy for listening frequency.
- Return list of dicts with keys: `rank` (1-based), `name`, `artist_name`, `album_name`, `catalog_id`.
- Use `datetime.now(UTC)` for cutoff calculation. Parse date_added_to_library strings to datetime if needed.

**`def compute_apple_music_top_artists(songs: list[dict[str, Any]], *, period: str = "month") -> list[dict[str, Any]]:`**
- Per D-05: aggregate from filtered tracks.
- Apply same period filtering as above.
- Group by artist_name, count tracks per artist, sort by count descending.
- Return list of dicts with keys: `rank` (1-based), `name`, `genres` (union of all genres from that artist's tracks), `track_count`.

All functions must have:
- `from __future__ import annotations` at top
- Type hints on all parameters and return types
- Docstrings with Args/Returns sections
- Proper logging with logger.info for counts

**`class StatsService:`**

Claude's discretion: Use a class (consistent with TasteService pattern). No caching -- stats are computed on-demand since the Spotify API already handles period filtering natively. For Apple Music, we re-fetch library data (could use TasteService's cached data if available, but simpler to fetch fresh since Apple library rarely exceeds 500 items).

Methods:

**`async def get_top_tracks(self, engine, encryption, settings, *, user_id: str, service: str | None = None, period: str = "month", limit: int = 20) -> dict[str, Any]:`**
- Call `_resolve_service(engine, user_id=user_id, service=service)` to get the target service.
- Call `_get_access_token(engine, encryption, settings, user_id=user_id, service=resolved_service)` to get a valid access token (with Spotify token refresh if needed).
- If Spotify: call `fetch_spotify_top_tracks_for_period(access_token, period=period, limit=limit)`. Return dict with keys: `service`, `period`, `items` (list of track dicts), `total` (len of items).
- If Apple Music: generate developer_token, call `fetch_apple_music_library(developer_token, access_token)`, then `compute_apple_music_top_tracks(songs, period=period)`, slice to limit. Return same structure.
- Raise ValueError if no connection found.

**`async def get_top_artists(self, engine, encryption, settings, *, user_id: str, service: str | None = None, period: str = "month", limit: int = 20) -> dict[str, Any]:`**
- Same pattern. Spotify: `fetch_spotify_top_artists_for_period(access_token, period=period, limit=limit)`.
- Apple Music: fetch library, `compute_apple_music_top_artists(songs, period=period)`, slice to limit.
- Return dict with `service`, `period`, `items`, `total`.

**`async def get_top_genres(self, engine, encryption, settings, *, user_id: str, service: str | None = None, period: str = "month", limit: int = 20) -> dict[str, Any]:`**
- Per D-04: genres derived from top artists' genre lists for Spotify.
- Spotify: fetch top artists via `fetch_spotify_top_artists_for_period`, then aggregate genres. For each artist, each genre gets +1 count per artist that has it, and track_count is derived from a separate top tracks fetch. Actually simpler: fetch top tracks AND top artists for the period. For each genre from artists, count how many unique artists have it (artist_count) and how many tracks (by cross-referencing track artist_names with artist genres) have it (track_count). Then rank by track_count descending.
  - Implementation detail: Build artist_name -> genres mapping from artists. For each track, look up its artist's genres. Tally genre -> track_count and genre -> set of artist_names. Sort by track_count desc. This is more accurate than just counting artist genres.
- Apple Music: songs already carry genre_names. Filter by period (same as top tracks), aggregate: genre -> track_count (how many tracks have this genre), genre -> artist_count (unique artists with this genre). Sort by track_count desc.
- Return dict with `service`, `period`, `items` (list of genre dicts with rank, genre, track_count, artist_count), `total`.

**`async def _resolve_service(self, engine, *, user_id: str, service: str | None) -> str:`**
- Same as TasteService._resolve_service: if service provided return it, else query get_user_connections and pick first.

**`async def _get_access_token(self, engine, encryption, settings, *, user_id: str, service: str) -> str:`**
- Query service_connections for this user+service.
- Decrypt access_token.
- If Spotify: check token_expires_at, refresh if within 60s of expiry (same pattern as TasteService._fetch_spotify_data).
- Return the (possibly refreshed) plaintext access_token.
- Raise ValueError if no connection found.

All methods must have full docstrings with Args/Returns, type hints, and proper logging. Use `from __future__ import annotations` at top.
