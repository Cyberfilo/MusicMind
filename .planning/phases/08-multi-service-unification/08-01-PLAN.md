# Phase 08 Plan 01: Genre Normalization, Track Deduplication, and Unified Profile

## Objective

Build the cross-service unification layer: a canonical genre taxonomy that normalizes Spotify and Apple Music genres, ISRC-based track deduplication with fuzzy title+artist fallback, and a unified taste profile pipeline that merges data from both connected services into a single coherent view.

## Context

- @backend/src/musicmind/engine/profile.py -- build_taste_profile, build_genre_vector consume genre lists
- @backend/src/musicmind/engine/similarity.py -- genre_jaccard uses expand_genres for Jaccard overlap
- @backend/src/musicmind/db/schema.py -- song_metadata_cache with isrc, service_source columns
- @backend/src/musicmind/api/taste/service.py -- TasteService fetches per-service data
- @backend/src/musicmind/api/recommendations/service.py -- RecommendationService uses single-service discovery

## Tasks

### Task 1: Genre taxonomy normalizer module
type="auto"

Create `backend/src/musicmind/engine/genres.py` -- a canonical genre mapping that normalizes both Spotify and Apple Music genre strings to shared canonical forms. Spotify genres are lowercase-hyphenated ("hip-hop", "italian-hip-hop"), Apple Music genres use Title Case with slashes ("Hip-Hop/Rap", "Italian Hip-Hop/Rap").

Implementation:
- Build a `CANONICAL_MAP` dict mapping known variants to canonical forms (e.g., "hip-hop" -> "Hip-Hop/Rap", "r-n-b" -> "R&B/Soul", "italian-hip-hop" -> "Italian Hip-Hop/Rap")
- `normalize_genre(genre: str) -> str` -- case-insensitive lookup, returns canonical or original
- `normalize_genre_list(genres: list[str]) -> list[str]` -- deduplicated normalized list
- `merge_genre_vectors(vector_a: dict[str, float], vector_b: dict[str, float]) -> dict[str, float]` -- weighted merge of two genre vectors with canonical normalization, re-normalize to sum=1.0
- Unit tests in `backend/tests/test_genre_normalizer.py`

Done: normalize_genre maps Spotify-style and Apple-style genre strings to a shared canonical form; merge_genre_vectors merges two vectors correctly.

### Task 2: Track deduplication module
type="auto"

Create `backend/src/musicmind/engine/dedup.py` -- cross-service track deduplication using ISRC primary key matching and fuzzy title+artist fallback.

Implementation:
- `deduplicate_tracks(tracks: list[dict]) -> list[dict]` -- dedup by ISRC first, then fuzzy title+artist for remaining non-ISRC tracks
- `_isrc_match(tracks: list[dict]) -> tuple[list[dict], list[dict]]` -- group by ISRC, merge service metadata, return (unique tracks, tracks without ISRC)
- `_fuzzy_title_artist_match(title_a: str, artist_a: str, title_b: str, artist_b: str) -> bool` -- normalize whitespace/case, strip parenthetical suffixes, compare with threshold
- When merging duplicates, prefer the record with more metadata (e.g., if Apple Music has genres, keep those)
- Unit tests in `backend/tests/test_dedup.py`

Done: deduplicate_tracks correctly merges cross-service duplicates via ISRC, with fuzzy fallback for tracks missing ISRC.

### Task 3: Unified taste profile service
type="auto"

Extend `TasteService` in `backend/src/musicmind/api/taste/service.py` to support `service="unified"` that fetches from both connected services, deduplicates songs, normalizes genres, and builds a merged profile.

Implementation:
- Add `_build_unified_profile()` method that: (a) fetches data from both services, (b) deduplicates songs via dedup module, (c) normalizes genres via genre normalizer, (d) calls build_taste_profile on merged data, (e) saves snapshot with service_source="unified"
- Modify `_resolve_service()` to accept "unified" and return it
- When service is None and user has both services connected, default to "unified"
- Update `get_profile()` to route to unified path when service == "unified"
- Add unified-specific response fields: `services_included: list[str]` to schemas
- Unit tests in `backend/tests/test_unified_taste.py` covering: both services fetched, dedup applied, genres normalized, single-service fallback still works

Done: TasteService supports service="unified" producing a merged profile from both services with proper dedup and genre normalization.

### Task 4: Cross-service recommendation support and integration tests
type="auto"

Wire unified profile into RecommendationService so recommendations draw from both catalogs.

Implementation:
- Update `RecommendationService.get_recommendations()` to detect unified profile and run discovery against both services in parallel
- Update `_resolve_credentials()` to return credentials for all connected services when unified
- Deduplicate recommendation candidates across services using dedup module
- Integration tests in `backend/tests/test_unified_recommendations.py` covering: unified recommendations use both services, cross-service dedup applied, single-service still works
- Run full test suite to ensure 146 existing tests plus new tests all pass

Done: RecommendationService draws from both service catalogs when user has unified profile; all existing and new tests pass.
