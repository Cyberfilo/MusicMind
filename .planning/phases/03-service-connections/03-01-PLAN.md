**Pure functions (no class needed for these):**

1. `generate_pkce_pair() -> tuple[str, str]` — Per D-01 and research Pattern 1. Use `secrets.token_urlsafe(64)` for code_verifier, `hashlib.sha256` + `base64.urlsafe_b64encode` with `rstrip(b"=")` for code_challenge. Returns `(code_verifier, code_challenge)`.

2. `build_spotify_authorize_url(client_id: str, redirect_uri: str, code_challenge: str, state: str) -> str` — Per research Pattern 2. Uses `urllib.parse.urlencode` with params: `response_type=code`, `client_id`, `scope=SPOTIFY_SCOPES` (per D-04), `redirect_uri`, `state`, `code_challenge_method=S256`, `code_challenge`. Returns full URL string.

3. `generate_apple_developer_token(team_id: str, key_id: str, private_key_path: str) -> str` — Per D-08 and research Pattern 5. Ported from `src/musicmind/auth.py` `_generate_developer_token()`. Read private key from `Path(private_key_path).expanduser().read_text().strip()`. Use `jwt.encode(payload, private_key, algorithm="ES256", headers=headers)` where payload = `{"iss": team_id, "iat": now, "exp": now + APPLE_TOKEN_EXPIRY_SECONDS}` and headers = `{"alg": "ES256", "kid": key_id}`. Import jwt as `import jwt as pyjwt` to avoid name collision with the auth jwt import pattern.

**Async functions (Spotify HTTP operations):**

4. `async def exchange_spotify_code(code: str, code_verifier: str, redirect_uri: str, client_id: str) -> dict` — Per D-02 and research Pattern 3. POST to SPOTIFY_TOKEN_URL with form data: `grant_type=authorization_code`, `code`, `redirect_uri`, `client_id`, `code_verifier`. Use `async with httpx.AsyncClient() as client`. Call `resp.raise_for_status()`. Return `resp.json()` which has `access_token`, `token_type`, `expires_in`, `refresh_token`, `scope`. CRITICAL: No `client_secret` in PKCE exchange (research anti-pattern 3).

5. `async def refresh_spotify_token(refresh_token_value: str, client_id: str) -> dict | None` — Per D-14, D-15 and research Pattern 4. POST to SPOTIFY_TOKEN_URL with form data: `grant_type=refresh_token`, `refresh_token=refresh_token_value`, `client_id`. CRITICAL: No `client_secret` for PKCE refresh (research Pitfall 3). On success (2xx), return `resp.json()`. On 4xx, return `None` (caller marks connection as expired).

6. `async def fetch_spotify_user_profile(access_token: str) -> dict` — GET SPOTIFY_ME_URL with `Authorization: Bearer {access_token}` header. Returns JSON with `id`, `email`, `display_name` fields. Used to get `service_user_id` on connect.

7. `async def check_apple_music_token(music_user_token: str, developer_token: str) -> bool` — Per D-16 and research Pattern 6. GET APPLE_MUSIC_STOREFRONT_URL with headers `Authorization: Bearer {developer_token}` and `Music-User-Token: {music_user_token}`. Return `True` if status is 200, `False` if status is 401 or 403 (per research Pitfall 5 — both codes mean expired). Handle exceptions gracefully (return False on connection errors).

**Async DB operations:**

8. `async def upsert_service_connection(engine, encryption: EncryptionService, *, user_id: str, service: str, access_token: str, refresh_token: str | None, expires_in: int | None, service_user_id: str | None) -> None` — Cross-dialect upsert using SELECT-then-INSERT/UPDATE within a transaction. Do NOT use `from sqlalchemy.dialects.postgresql import insert as pg_insert` — that is PostgreSQL-specific and breaks SQLite test infrastructure. Instead, implement as:

```python
async with engine.begin() as conn:
    # Check if row already exists for this user+service
    existing = await conn.execute(
        sa.select(service_connections.c.id).where(
            sa.and_(
                service_connections.c.user_id == user_id,
                service_connections.c.service == service,
            )
        )
    )
    row = existing.first()

    encrypted_access = encryption.encrypt(access_token)
    encrypted_refresh = encryption.encrypt(refresh_token) if refresh_token else None
    token_expires_at = (
        datetime.now(UTC) + timedelta(seconds=expires_in) if expires_in else None
    )
    now = datetime.now(UTC)

    if row:
        # UPDATE existing connection
        await conn.execute(
            service_connections.update()
            .where(service_connections.c.id == row.id)
            .values(
                access_token_encrypted=encrypted_access,
                refresh_token_encrypted=encrypted_refresh,
                token_expires_at=token_expires_at,
                service_user_id=service_user_id,
                connected_at=now,
            )
        )
    else:
        # INSERT new connection
        await conn.execute(
            service_connections.insert().values(
                user_id=user_id,
                service=service,
                access_token_encrypted=encrypted_access,
                refresh_token_encrypted=encrypted_refresh,
                token_expires_at=token_expires_at,
                service_user_id=service_user_id,
                connected_at=now,
            )
        )
```

This pattern is dialect-agnostic — works on both PostgreSQL (production) and SQLite (tests). The `engine.begin()` context manager ensures the SELECT and INSERT/UPDATE run in a single transaction, providing the same atomicity guarantee as `ON CONFLICT DO UPDATE`.

9. `async def delete_service_connection(engine, *, user_id: str, service: str) -> bool` — Per D-13. DELETE from `service_connections` WHERE user_id and service match. Return True if a row was deleted, False if no matching row. NOTE: Spotify has no revocation API (research Pitfall 7) — just delete the row.

10. `async def get_user_connections(engine, *, user_id: str) -> list[dict]` — SELECT all columns from `service_connections` WHERE user_id matches. Return list of row dicts with keys: `service`, `access_token_encrypted`, `refresh_token_encrypted`, `token_expires_at`, `service_user_id`, `connected_at`. The router will derive status from `token_expires_at` (per D-12, research Pitfall 6 — no external API calls on status check).

All async DB operations use `async with engine.begin() as conn:` pattern (matches auth/router.py).
All imports: `from __future__ import annotations`, `import base64`, `import hashlib`, `import logging`, `import secrets`, `import time`, `import urllib.parse`, `from datetime import UTC, datetime, timedelta`, `from pathlib import Path`, `import httpx`, `import jwt as pyjwt`, `import sqlalchemy as sa`, `from musicmind.db.schema import service_connections`, `from musicmind.security.encryption import EncryptionService`.

NOTE: No `from sqlalchemy.dialects.postgresql import insert` — removed to ensure cross-dialect compatibility with SQLite test infra.

Add `logger = logging.getLogger(__name__)` and log key operations (token exchange, refresh, health check).
