**File 1: `backend/src/musicmind/api/services/router.py`**

Create with `from __future__ import annotations`, module docstring, and these 6 endpoints. Follow the auth/router.py pattern exactly: `router = APIRouter(prefix="/api/services", tags=["services"])`, access `request.app.state.engine`, `request.app.state.settings`, `request.app.state.encryption`. All endpoints require `current_user: dict = Depends(get_current_user)` except the Spotify callback (which uses session-stored user context — see below).

Add `logger = logging.getLogger(__name__)` per conventions.

**Endpoint 1: GET /api/services** (per D-11, D-12 — SVCN-04)
```python
@router.get("")
async def list_connections(request: Request, current_user: dict = Depends(get_current_user)):
```
- Call `get_user_connections(engine, user_id=current_user["user_id"])`
- Build response for BOTH services ("spotify" and "apple_music"):
  - If connection row exists for a service: check `token_expires_at` — if it's in the past, status = "expired"; otherwise status = "connected". For Apple Music (where `token_expires_at` is NULL), status = "connected" (health check happens at API call time per Pitfall 6).
  - If no connection row: status = "not_connected", service_user_id = None, connected_at = None
- Return `ServiceListResponse` with both services always listed (per D-12)
- IMPORTANT: No external API calls here (per research Pitfall 6 — status endpoint must be fast, DB-only)

**Endpoint 2: POST /api/services/spotify/connect** (per D-01, D-02 — SVCN-01)
```python
@router.post("/spotify/connect")
async def spotify_connect(request: Request, current_user: dict = Depends(get_current_user)):
```
- Verify `settings.spotify_client_id` is not None, raise 400 if missing ("Spotify not configured")
- Call `generate_pkce_pair()` to get `(code_verifier, code_challenge)`
- Generate `state = secrets.token_urlsafe(16)`
- Store in session (per D-01 and research Pattern 2):
  ```python
  request.session["spotify_code_verifier"] = code_verifier
  request.session["spotify_state"] = state
  request.session["spotify_user_id"] = current_user["user_id"]
  ```
  NOTE: Store `user_id` in session because the callback endpoint cannot use `get_current_user` dependency (Spotify redirects the browser — no access_token cookie on redirect back from spotify.com)
- Call `build_spotify_authorize_url(settings.spotify_client_id, settings.spotify_redirect_uri, code_challenge, state)`
- Return `SpotifyConnectResponse(authorize_url=url)`

**Endpoint 3: GET /api/services/spotify/callback** (per D-02 — SVCN-01)
```python
@router.get("/spotify/callback")
async def spotify_callback(request: Request, code: str, state: str):
```
NOTE: This endpoint does NOT use `Depends(get_current_user)`. The browser is redirected here by Spotify — there are no auth cookies on this request. Instead, the user_id is retrieved from the session (stored during connect).

- Verify `state` matches `request.session.get("spotify_state")` — raise 400 "Invalid state" if mismatch (this IS the CSRF protection for OAuth, per research Open Question 3)
- Retrieve `code_verifier = request.session.get("spotify_code_verifier")` and `user_id = request.session.get("spotify_user_id")` — raise 400 if either is missing
- Clear session keys after retrieval: `request.session.pop("spotify_code_verifier", None)`, same for state and user_id
- Call `exchange_spotify_code(code, code_verifier, settings.spotify_redirect_uri, settings.spotify_client_id)`
- Extract `access_token`, `refresh_token`, `expires_in` from response
- Call `fetch_spotify_user_profile(access_token)` to get `service_user_id = profile["id"]`
- Call `upsert_service_connection(engine, encryption, user_id=user_id, service="spotify", access_token=access_token, refresh_token=refresh_token, expires_in=expires_in, service_user_id=service_user_id)` (per D-03)
- logger.info("Spotify connected for user: %s", user_id)
- Return `{"message": "Spotify connected", "service_user_id": service_user_id}`
- Wrap token exchange and profile fetch in try/except httpx.HTTPStatusError — raise 400 with detail from Spotify error on failure

**Endpoint 4: GET /api/services/apple-music/developer-token** (per D-08 — SVCN-02)
```python
@router.get("/apple-music/developer-token")
```

**Endpoint 5: POST /api/services/apple-music/connect** (per D-07 — SVCN-02)

- logger.info("Apple Music connected for user: %s", current_user["user_id"])
- Return `{"message": "Apple Music connected"}`

**Endpoint 6: DELETE /api/services/{service}** (per D-13 — SVCN-03)
```python
@router.delete("/{service}")
async def disconnect_service(request: Request, service: str, current_user: dict = Depends(get_current_user)):
```
- Validate `service` is one of `("spotify", "apple_music")` — raise 400 if not
- Call `delete_service_connection(engine, user_id=current_user["user_id"], service=service)`
- If returned False (no row deleted), raise 404 "Service not connected"
- logger.info("Service %s disconnected for user: %s", service, current_user["user_id"])
- Return `DisconnectResponse(message=f"{service} disconnected")`
- NOTE: Per research Pitfall 7, Spotify has no revocation API — deletion only. Add comment in code.
