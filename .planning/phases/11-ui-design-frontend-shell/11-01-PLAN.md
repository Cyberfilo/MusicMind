<interfaces>
<!-- Backend auth API contract the frontend must call -->

POST /api/auth/signup
  Request: { email: string, password: string (min 8), display_name?: string }
  Response 201: { user_id: string, email: string, message: "success" }
  Sets: access_token and refresh_token as httpOnly cookies

POST /api/auth/login
  Request: { email: string, password: string }
  Response 200: { user_id: string, email: string, message: "success" }
  Sets: access_token and refresh_token as httpOnly cookies

POST /api/auth/logout
  Response 200: { message: "Logged out" }
  Clears: access_token and refresh_token cookies

POST /api/auth/refresh
  Response 200: { message: "Token refreshed" }
  Requires: valid refresh_token cookie
  Sets: new access_token cookie

GET /api/auth/me
  Response 200: { user_id: string, email: string, display_name: string }
  Requires: valid access_token cookie

All POST/PUT/DELETE endpoints require CSRF token:
  - GET any endpoint first to receive csrftoken cookie
  - Send X-CSRFToken header with the csrftoken cookie value on mutating requests
</interfaces>

    5. Create `frontend/src/lib/api.ts` -- central API client:
       - Export `const API_URL = process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000"`
       - Export `async function apiFetch<T>(path: string, options?: RequestInit): Promise<T>` that:
         a. Prepends API_URL to path
         b. Always includes `credentials: "include"` (for httpOnly cookies)
         c. Always includes `Content-Type: application/json` for POST/PUT/DELETE
         d. On POST/PUT/DELETE: reads `csrftoken` from `document.cookie` and adds `X-CSRFToken` header
         e. If response is 401, attempts POST /api/auth/refresh, retries once, then redirects to /login
         f. Throws on non-ok response with parsed error message
       - Export typed wrapper functions:
         - `authApi.signup(email, password, displayName?)` -> POST /api/auth/signup
         - `authApi.login(email, password)` -> POST /api/auth/login
         - `authApi.logout()` -> POST /api/auth/logout
         - `authApi.me()` -> GET /api/auth/me

    6. Create `frontend/src/stores/auth-store.ts` -- Zustand store per D-03:
       - State: `{ user: { user_id, email, display_name } | null, isLoading: boolean, isAuthenticated: boolean }`
       - Actions: `setUser(user)`, `clearUser()`, `checkAuth()` (calls /api/auth/me, sets user or clears)

    7. Create `frontend/src/lib/auth.ts`:
       - Export `async function requireAuth()` -- calls auth store checkAuth, returns user or redirects to /login
       - Export helper to read CSRF token from cookies

    8. Create `frontend/src/providers/query-provider.tsx` -- TanStack Query provider:
