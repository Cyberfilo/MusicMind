    1. Create backend/src/musicmind/api/chat/tools.py with 8 curated tool definitions (per D-04, D-05):

       TOOL_DEFINITIONS: list of Anthropic tool schemas. Each tool is a dict with:
       - name: str
       - description: str (what Claude sees -- be specific about MusicMind capabilities)
       - input_schema: JSON Schema dict (type: object, properties, required)

       The 8 tools (per D-04):
       a) get_taste_profile -- calls TasteService.get_profile. Input: optional service (str), optional force_refresh (bool). Returns user's genre vector, top artists, audio trait preferences.
       b) get_recommendations -- calls RecommendationService.get_recommendations. Input: optional strategy (str), optional mood (str), optional limit (int). Returns scored recommendations with explanations.
       c) get_listening_stats_tracks -- calls StatsService.get_top_tracks. Input: optional period (str: month/6months/alltime), optional limit (int). Returns top tracks by time period.
       d) get_listening_stats_artists -- calls StatsService.get_top_artists. Input: optional period (str), optional limit (int). Returns top artists by time period.
       e) get_top_genres -- calls StatsService.get_top_genres. Input: optional period (str), optional limit (int). Returns top genres by time period.
       f) give_feedback -- calls RecommendationService.submit_feedback. Input: catalog_id (str, required), feedback_type (str: thumbs_up/thumbs_down/skip, required). Records user feedback on a recommendation.
       g) get_recommendations_by_description -- calls RecommendationService.get_recommendations with mood parameter. Input: description (str, required). Claude interprets the description and maps it to an appropriate mood/strategy combination.
       h) adjust_taste_preferences -- a meta-tool that calls get_recommendations with specific strategy/mood based on Claude's interpretation. Input: adjustment (str, required -- e.g., "less mainstream pop", "more electronic"). Returns recommendations reflecting the adjustment.

       TOOL_EXECUTORS: dict[str, Callable] mapping tool name -> async executor function.
       Each executor function signature: async def execute_tool_name(*, engine, encryption, settings, user_id, **kwargs) -> dict
       The executor functions are thin wrappers that call the appropriate service method.
       For tools g and h: these map to RecommendationService.get_recommendations with derived strategy/mood params. The mapping logic is simple -- Claude provides the natural language, and the executor passes reasonable defaults.
