    familiarity_score: float = Field(description="0=focused on few genres, 1=adventurous (Shannon entropy)")
    genre_vector: dict[str, float] = Field(description="Genre name -> normalized affinity")

Create `backend/src/musicmind/api/taste/fetch.py` with data fetching functions for both services. All functions are async, use httpx, and return lists of dicts compatible with song_metadata_cache column shape (per D-05).

**File header:**
```python
"""Data fetching functions for Spotify and Apple Music taste profile pipeline.

Fetches library/top/recently-played data from each service and returns
song_metadata_cache-compatible dicts. Genre enrichment for Spotify uses
the top_artists response (Spotify tracks do NOT include genres).
"""
from __future__ import annotations

import logging
from typing import Any

import httpx

logger = logging.getLogger(__name__)

SPOTIFY_API_BASE = "https://api.spotify.com/v1"
APPLE_MUSIC_API_BASE = "https://api.music.apple.com/v1"
```

**Spotify functions (per D-05, Pitfall 1, Pitfall 4, Pitfall 5):**

1. `_spotify_track_to_cache_dict(track: dict) -> dict[str, Any]`:
   - Maps Spotify track object fields to song_metadata_cache columns exactly as in RESEARCH.md code example
   - genre_names starts as [] (populated separately from artist genres per Pitfall 1)
   - service_source = "spotify"
   - Handle nested access safely (track.get("album", {}), track.get("artists", []), track.get("external_ids", {}))

2. `async def fetch_spotify_top_tracks(access_token: str, *, limit: int = 50) -> list[dict[str, Any]]`:
   - GET /v1/me/top/tracks?time_range=long_term&limit={limit}&offset=0
   - Headers: Authorization: Bearer {access_token}
   - Paginate with offset: fetch up to 200 total (4 pages of 50)
   - Return list of _spotify_track_to_cache_dict results
   - On 403 (dev mode restriction), log warning and return empty list (fallback per RESEARCH.md)

3. `async def fetch_spotify_top_artists(access_token: str, *, limit: int = 50) -> list[dict[str, Any]]`:
   - GET /v1/me/top/artists?time_range=long_term&limit={limit}&offset=0
   - Returns list of artist dicts with keys: {"id": str, "name": str, "genres": list[str]}
   - Paginate up to 100 artists (2 pages)
   - This is the GENRE SOURCE for Spotify (Pitfall 1)

4. `async def fetch_spotify_saved_tracks(access_token: str, *, limit: int = 50, max_pages: int = 4) -> list[dict[str, Any]]`:
   - GET /v1/me/tracks?limit={limit}&offset=0
   - Paginate up to max_pages (default 200 tracks)
   - Map each item via _spotify_track_to_cache_dict(item["track"])
   - Also extract date_added_to_library from item["added_at"]

5. `async def fetch_spotify_recently_played(access_token: str, *, limit: int = 50) -> list[dict[str, Any]]`:
   - GET /v1/me/player/recently-played?limit={limit}
   - Returns list of history-shaped dicts: {"song_id": track_id, "song_name": name, "artist_name": artist, "album_name": album, "genre_names": [], "duration_ms": int, "observed_at": played_at, "service_source": "spotify"}

6. `def enrich_spotify_genres(tracks: list[dict[str, Any]], artists: list[dict[str, Any]]) -> None`:
   - Builds artist_name -> genres mapping from artists list
   - For each track in tracks, sets track["genre_names"] = artist_genres.get(track["artist_name"], [])
   - Mutates tracks in-place. Per Pitfall 1: this is the ONLY way Spotify tracks get genres.

**Apple Music functions (per D-05, Pitfall 3, Pitfall 5):**

7. `async def fetch_apple_music_library(developer_token: str, music_user_token: str, *, limit: int = 100, max_pages: int = 5) -> list[dict[str, Any]]`:
   - GET /v1/me/library/songs?limit={limit}&include[library-songs]=catalog
   - Headers: Authorization: Bearer {developer_token}, Music-User-Token: {music_user_token}
   - BOTH tokens required (Pitfall 3)
   - Paginate up to max_pages (default 500 songs, per Pitfall 5 cap)
   - Map each resource to cache dict: catalog_id from resource.attributes.playParams.catalogId or resource.id, name, artist_name (artistName), album_name (albumName), genre_names (genreNames), duration_ms (durationInMillis), release_date (releaseDate), audio_traits (audioTraits or []), date_added_to_library (dateAdded), service_source = "apple_music"
   - Use ?include[library-songs]=catalog to get catalog metadata (genres, ISRC, audio traits)

8. `async def fetch_apple_music_recently_played(developer_token: str, music_user_token: str, *, limit: int = 10, max_pages: int = 5) -> list[dict[str, Any]]`:
   - GET /v1/me/recent/played/tracks?limit={limit}
   - Returns history-shaped dicts similar to Spotify recently_played
   - Paginate up to max_pages (50 tracks)

**All functions must:**
- Use `async with httpx.AsyncClient() as client:` per established pattern
- Handle httpx.HTTPStatusError: log error, return empty list (never crash)
- Use `%-formatting` in logger calls (convention)
- Have return type annotations including -> None where applicable
- Have docstrings
